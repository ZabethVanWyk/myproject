FEATURE CARDS COMPONENT

OVERVIEW

This component replaces image-based feature cards with a reusable HTML, CSS and JavaScript solution.

WHY THIS APPROACH

The original implementation required image editing whenever statistics changed.

This solution stores content as data and renders cards dynamically, making updates faster and easier.

ACCESSIBILITY

- Semantic HTML structure
- Uses article elements for content grouping
- Responsive layout
- Screen reader friendly section labels

CMS INTEGRATION

The card content is stored in a JavaScript data structure.

This allows future integration with:
- WordPress
- Drupal
- Umbraco
- Sitecore
- Headless CMS APIs

RESPONSIVE DESIGN

Cards display in three columns on desktop and stack vertically on smaller screens.

TECHNOLOGIES USED

- HTML5
- CSS3
- Vanilla JavaScript
